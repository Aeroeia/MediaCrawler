# 跨仓库对话软链接协议（MediaCrawler + wechat-spider）

## 1. 目标

- 在不改代码的前提下，让对话同时加载两个仓库上下文：
  - `/Users/aer/IdeaProjects/MediaCrawler`
  - `/Users/aer/IdeaProjects/wechat-spider`
- 用于方案评审、集成设计、字段映射讨论。

## 2. 结论与边界

- 结论：当前阶段不需要安装额外 skill，也不需要改仓库结构。
- 边界：仅建立“认知关联”，不做仓库合并、不做依赖打通、不做接口改造。
- 触发词：当用户明确说“开始一期实现”后，才进入工程改造计划。

## 3. 会话级关联规则（默认执行）

每次新需求建议使用以下输入模板：

```text
工作目录：
- /Users/aer/IdeaProjects/MediaCrawler
- /Users/aer/IdeaProjects/wechat-spider

本次请同时参考两个项目实现。
任务类型：控制面 / 数据面（两者选一，或同时）
目标：<你的目标>
约束：当前先不改代码，只做方案评审（如适用）
```

补充说明：
- 控制面：启动、停止、日志、任务编排、运行管理。
- 数据面：字段映射、存储落库、模型对齐、清洗规则。

## 5. 对话验收标准

- 场景 1：提“接入 MediaCrawler 控制面”时，回答必须同时引用两个项目入口并给出整合路径。
- 场景 2：提“对齐微信文章字段到 MediaCrawler 存储”时，回答必须给出字段映射建议并指出差异。
- 场景 3：只提一个仓库时，先确认是否需要同时参考另一个仓库，避免遗漏上下文。

## 6. 何时再考虑安装 skill

仅当你需要“固定化、一键化流程”时再考虑，例如：
- 固定生成 `requirements/design/tasks` 文档链路；
- 固定代码审查模板与输出格式；
- 固定多项目联动执行脚本。

当前“软链接认知”阶段，skill 不是前置条件。
